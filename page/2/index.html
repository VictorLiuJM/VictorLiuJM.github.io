递归和栈的相关性分析

递归的基本思想是将一个规模较大的问题分解为规模较小的多个子问题, 而每一个子问题都可以拆解成多个更小的子问题. 最重要的是子问题可以解决, 然后基于子问题从而解决更大的问题.

所以递归解决的是有依赖顺序关系的多个问题.

假设依赖关系如下 :

[A] ——依赖——> [B] ——依赖 ——> [C]

由于A、B、C 之间有依赖关系, 所以要解决问题 A, 需要先解决问题 B; 要解决问题 B, 需要先解决问题 C; 所以这个问题的解决顺序如下 :

先解决 C, 然后再解决 B, 最后解决 A.

从函数调用看广义递归

函数的调用就是一个广义的递归过程 :

具体的调用过程如下 :

调用函数 A;

调用函数 B;

调用函数 C;

函数 C 返回;

函数 B 返回;

函数 A 返回;

狭义递归函数

当上面的 A, B, C 都是同一个函数时, 即函数内调用函数自身. 递归对问题的处理, 遵循了先进后出的原则, 和栈的原则是一致的.

使用递归函数时, 最大的问题就是当递归函数的递归次数很大的时候, 会导致函数栈溢出而死机. 每当调用一个函数时, 在这个函数执行之前都会将之前的代码地址 (也就是调用点) 入栈, 等被调用的函数执行完之后将地址出栈, 程序根据这个数据返回调用点. 当递归次数多的时候, 只会入栈不会出栈, 于是堆栈就被挤爆了. 一般操作系统分配给一个进程的栈空间是 2M.
